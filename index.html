2.1 DQL Abfrage Sprache

vertikale Selektion

Damit beschränkt man die Attribute die beim Ergebnis der Datensätze angezeigt werden sollen

horizontale Selektion

horizontale Selektion beschränkt die Ergebnismenge der Datensätze mittels eines Kriteriumausdruckes der erfüllt sein muss.

SQL "Structured Query Language“ strukturierte Abfragesprache

DQL Data Query Language -> Abragen von Daten
DML Data Manipolation Language -> Daten einfügen, bearbeiten, löschen
DDL Data Definition Language -> Tabellen festlegen, bearbeiten, Datenbank anlegen
DCL Data Control Language -> Rechtevergabe innerhalb der Datenbank

DQL - Abfragesprache

SELECT *

  • alle Spalten der Tabelle

Attribut 1, Attribut 2, Attribut 3

  • nur jene Spalten (vertikale Selektion der Tabelle)
  • Beschränkung der Spalten die angezeigt werden sollen

FROM Tabellenname

  • von wo die Daten geladen werden sollen, können auch mehrere Tabellen sein
  • Präfix notwendig

WHERE Spalte (Attribut1) Operator Bedingung

  • Vergleich einer Spalte oder mehrere Spalten mit einer Bedingung.
  • Einschränkung der Datensätze die die Bedingung erfüllen
  • horizontale Selektion

Wir unterscheiden die vertikale und horizontale Selektion

  • vertikal: Beschränkung der Attribute(Spalten) die im Ergebnis der Abfrage für den User angezeigt werden sollen
  • horizontal: Beschränkung der Datensätze die dem User beim Ergebnis angezeigt werden sollen. Kriterium / die Bedingung muss erfüllt sein ansonsten wird der Datensatz nicht in das Ergebnis aufgenommen. 

Bedingungen bei SELECT

Vergleich: =

SELECT * FROM Schueler WHERE Alter = 12 --> ist eine Zahl kann ohne " " verglichen werden 
SELECT * FROM Schueler WHERE Name = 'Chris' --> ist ein Text hier benötigt man ' ' da es sich um einen String als Datentyp handelt.

Ungleich: <>

SELECT * FROM Schueler WHERE Alter <> 12 --> ist eine Zahl kann ohne " " verglichen werden

LIKE  

suchen nach einem bestimmten Muster innerhalb einer Attributspalte.

LIKE Operator Description
WHERE CustomerName LIKE 'a%' Finds any values that start with "a"
WHERE CustomerName LIKE '%a' Finds any values that end with "a"
WHERE CustomerName LIKE '%or%' Finds any values that have "or" in any position
WHERE CustomerName LIKE '_r%' Finds any values that have "r" in the second position
WHERE CustomerName LIKE 'a_%' Finds any values that start with "a" and are at least 2 characters in length
WHERE CustomerName LIKE 'a__%' Finds any values that start with "a" and are at least 3 characters in length
WHERE ContactName LIKE 'a%o' Finds any values that start with "a" and ends with "o" 
SELECT Vorname, Nachname, ...
FROM Schueler
WHERE Vorname LIKE 'a%';

alle Schüler die im Vornamen mit einem a beginnen.

IN Operator

Mittels des Operator wird ein Attribut auf eine Liste von zulässigen  Kriterien  geprüft.

SELECT Vorname, nachname
FROM Schueler
WHERE Ort IN (Krems, Wien, ...);

BETWEEN Operator

Mittels des Between Operators kann ein Bereich der abgefragt werden soll angeben werden. Mittels einer oberen und unteren Schranke

SELECT Vorname, Nachname
FROM Schueler
WHERE Alter BETWEEN 15 AND 17;

Min / Max Funktionen

Abfrage des  minimalen  und  maximalen  Betrag eines Attributes

SELECT MIN(Alter)
FROM Schueler
WHERE ORT ='Wien';

SELECT MAX(Alter)
FROM Schueler
WHERE ORT ='Wien';

DISTINCT

Doppelte Werte werden beseitigt. 

SELECT DISTINCT column1, column2, ...
FROM table_name;

Unterschied zwischen Group BY und DISTINCT?

DISTINCT lässt bei der Auswahl der Daten die doppelten Werte aus, GROUP BY fasst nach der Selektion die Daten zusammen lädt sie aber.

Logische Operatoren

AND

SELECT column1, column2, ...
FROM table_name
WHERE condition1 AND condition2 AND condition3 ...;

OR

SELECT column1, column2, ...
FROM table_name
WHERE condition1 OR condition2 OR condition3 ...;

NOT

SELECT column1, column2, ...
FROM table_name
WHERE NOT condition;

ORDER BY um Daten zu sortieren

ASC aufsteigend und DESC absteigend

SELECT column1, column2, ...
FROM table_name
ORDER BY column1, column2, ... ASC|DESC;

SELECT * FROM Customers
ORDER BY Country;

Gruppierungen

SELECT column_name(s)
FROM table_name
WHERE condition
GROUP BY column_name(s)
ORDER BY column_name(s);

SELECT COUNT(CustomerID), Country
FROM Customers
GROUP BY Country;

HAVING (=WHERE bei GROUP BY) bei virtuellen Tabellen 

WHERE kann nur auf gespeicherte Daten angewendet werden, MIN, MAX, AVG, SUM, COUNT sind virtuelle Tabellen für diese muss Having angewendet werden da WHERE für Bedingungen nicht funktioniert.

SELECT column_name(s)
FROM table_name
WHERE condition
GROUP BY column_name(s)
HAVING condition
ORDER BY column_name(s);

SELECT COUNT(CustomerID), Country
FROM Customers
GROUP BY Country
HAVING COUNT(CustomerID) > 5;

Erkennung von NULL Values

SELECT column_names
FROM table_name
WHERE column_name IS NULL;

SELECT column_names
FROM table_name
WHERE column_name IS NOT NULL;

Alias alternative Namen für eine Spalte:

SELECT column_name AS alias_name
FROM table_name;

Data Manipulation Language (DML)

INSERT Daten in die Datenbank einfügen

INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);

INSERT INTO Customers (CustomerName, ContactName, Address, City, PostalCode, Country)
VALUES ('Cardinal', 'Tom B. Erichsen', 'Skagen 21', 'Stavanger', '4006', 'Norway');

UPDATE Daten in der Datenbank bearbeiten

UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;

UPDATE Customers
SET ContactName = 'Alfred Schmidt', City= 'Frankfurt'
WHERE CustomerID = 1;

DELETE Daten innerhalb der Datenbank bearbeiten

DELETE FROM table_name WHERE condition;

DELETE FROM Customers WHERE CustomerName='Alfreds Futterkiste';
